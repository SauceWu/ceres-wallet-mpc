**sign_start implementation:**
1. Parse message_hash_hex via `MessageDigest::from_hex(&message_hash_hex)?` (type safety boundary)
2. Parse server_payload as WireEnvelope, validate `from_id == 1` (T-08-04 mitigation)
3. Deserialize keyshare: `let keyshare: dkls23_ll::dkg::Keyshare = serde_json::from_str(&share)?`
4. Initialize DSG State: `let chain_path = DerivationPath::from_str("m").unwrap();` then `dsg::State::new(&mut rng, keyshare, &chain_path).map_err(|e| e.to_string())?`
5. Call `state.generate_msg1()` (required to advance local state)
6. Decode server's SignMsg1 from envelope payload: `let server_msg1: dsg::SignMsg1 = decode_cbor_base64(&server_env.payload)?`
7. Call `state.handle_msg1(&mut rng, vec![server_msg1])` -> Vec<SignMsg2>
8. Encode msg2[0] as CBOR Base64, wrap in WireEnvelope(protocol=Dsg, round=2, from_id=0, to_id=Some(1), step=None)
9. Store SignSession { state, round: 2, digest, consumed: false, partial_sig: None, pending_msg4: None } in SIGN_SESSIONS
10. Return MpcRoundResult { status: "in_progress", round: 2, client_payload: Some(env_json) }

**sign_continue implementation — dispatch by session.round:**

**Round 2 (server sends SignMsg2):**
1. Validate from_id == 1
2. Get session from SIGN_SESSIONS (get_mut, not remove)
3. Decode server SignMsg2 from payload
4. Call `session.state.handle_msg2(&mut rng, vec![server_msg2])` -> Vec<SignMsg3>
5. Encode msg3[0] as WireEnvelope(round=3, P2P to=1)
6. Update session.round = 3
7. Return MpcRoundResult { status: "in_progress", round: 3 }

**Round 3 (server sends SignMsg3) — CRITICAL: SEC-01 enforcement:**
1. Validate from_id == 1
2. **REMOVE session from SIGN_SESSIONS** (not get_mut — this prevents re-entry, SEC-01)
3. Check `session.consumed == true` -> return Err("session already consumed")
4. Decode server SignMsg3
5. Call `session.state.handle_msg3(vec![server_msg3])` -> PreSignature (NOTE: no rng, no commitment_2_list — DSG is simpler than DKG)
6. **Immediately consume PreSignature:** `let (partial, msg4) = create_partial_signature(pre, session.digest.into_bytes())`
   - PreSignature is consumed by move here — Rust compiler prevents reuse
7. Encode msg4 as CBOR Base64
8. Create new session entry with consumed=true, partial_sig=Some(partial), pending_msg4=Some(msg4_cbor_bytes)
   - Re-insert into SIGN_SESSIONS for Round 4 retrieval
   - Or alternatively: store partial_sig + pending_msg4 in session, mark consumed, re-insert
9. Wrap msg4 in WireEnvelope(round=4, broadcast to_id=None)
10. session.round = 4, session.consumed = true
11. Re-insert session into SIGN_SESSIONS
12. Return MpcRoundResult { status: "in_progress", round: 4 }

**Round 4 (server sends SignMsg4):**
1. Validate from_id == 1
2. REMOVE session from SIGN_SESSIONS (final removal)
3. Decode server SignMsg4
4. Take partial_sig from session: `session.partial_sig.take().ok_or("partial_sig not set")?`
5. Call `combine_signatures(partial, vec![server_msg4])` -> Signature
6. **Calculate recid via trial recovery:**
   ```rust
   // Need the original keyshare's public_key for ecrecover
   // Problem: we don't have keyshare anymore (consumed in State::new)
   // Solution: extract VerifyingKey from signature + hash via RecoveryId::trial_recovery_from_prehash
   ```
   IMPORTANT: We need the VerifyingKey for trial_recovery. Since keyshare was consumed by State::new and we only have the session, we need to store the public key at sign_start time. **Add `pub public_key: k256::AffinePoint` field to SignSession** (or store serialized verifying key bytes). At sign_start, extract `keyshare.public_key` before passing keyshare to State::new, then store in session.

   Then at Round 4:
   ```rust
   let vk = VerifyingKey::from_affine(session.public_key)
       .map_err(|e| format!("invalid public key: {e}"))?;
   let recid = RecoveryId::trial_recovery_from_prehash(&vk, &digest_bytes, &sig)
       .map_err(|e| format!("recid recovery failed: {e}"))?;
   ```
7. Extract r, s: `let (r_bytes, s_bytes) = sig.split_bytes();`
8. Build SignCompletedPayload { r: hex::encode(r_bytes), s: hex::encode(s_bytes), recid: recid.to_byte() }
9. Return MpcRoundResult { status: "completed", round: 4, client_payload: Some(completed_json) }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server -> client | Server envelopes may contain malicious payload; from_id must be validated |
| client sign_start entry | message_hash_hex validated via MessageDigest::from_hex; keyshare JSON from local storage |
| PreSignature lifecycle | Must be consumed exactly once; nonce reuse = private key leak |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Spoofing | sign_start, sign_continue | mitigate | Validate `from_id == 1` on all incoming server envelopes (same pattern as keygen) |
| T-10-02 | Tampering | PreSignature reuse (nonce reuse -> key extraction) | mitigate | (1) Rust move semantics consume PreSignature in create_partial_signature; (2) session.consumed flag; (3) session removed from SIGN_SESSIONS after Round 3 entry, re-inserted only with consumed=true |
| T-10-03 | Tampering | DSG protocol message integrity | accept | dkls23-ll protocol layer validates cryptographic commitments internally (handle_msg2 checks final_session_id) |
| T-10-04 | Information Disclosure | Keyshare in memory during signing | accept | Keyshare consumed by State::new (moved into DsgState); session cleaned up after Round 4; full memory zeroization deferred to production hardening |
| T-10-05 | Denial of Service | Orphaned sign sessions | accept | Deferred to Phase 11 SEC-02 Session TTL; current risk: memory leak for abandoned sessions |
| T-10-06 | Tampering | Raw bytes as message digest | mitigate | MessageDigest::from_hex at sign_start boundary (SEC-03, already implemented in Phase 8) |
</threat_model>
