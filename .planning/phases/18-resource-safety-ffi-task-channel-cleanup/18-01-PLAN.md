In mpc_engine.rs recover() round>1 TTL check path (around lines 379-381), the current code removes the session but does not abort the task. Per D-03 and D-04:

Replace:
```rust
Some(s) if s.created_at.elapsed() > SESSION_TTL => {
    sessions.remove(&session_id);
    return Err(format!("recovery session expired (TTL): {session_id}"));
}
```

With:
```rust
Some(s) if s.created_at.elapsed() > SESSION_TTL => {
    let session = sessions.remove(&session_id).unwrap();
    if let Some(handle) = session.task_handle {
        handle.abort();
    }
    return Err(format!("recovery session expired (TTL): {session_id}"));
}
```

Note: `sessions` is already locked (`let mut sessions = RECOVERY_SESSIONS.lock().unwrap()`), so `remove()` is safe. The `unwrap()` on remove is safe because we just matched `Some(s)` on the same key. Use `if let Some(handle)` because task_handle is `Option<JoinHandle>` — it could be `None` if already taken.

- Normal completion: join task (existing, correct)
- inject_all failure: abort task (SAFE-01, this plan Task 1)
