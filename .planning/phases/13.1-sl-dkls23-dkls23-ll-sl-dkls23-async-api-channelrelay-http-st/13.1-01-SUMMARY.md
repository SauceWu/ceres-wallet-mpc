# Phase 13.1 Plan 01: Cargo Dep Swap + ChannelRelay Infrastructure Summary

**One-liner:** Replaced dkls23-ll with sl-dkls23 1.0.0-beta ecosystem, created ChannelRelayConn (Stream+Sink+Unpin+Relay) bridge and OnceCell tokio runtime singleton, rewrote session.rs to channel-based types.

## What Was Built

### Task 1: Cargo.toml + runtime.rs + relay.rs

**Cargo.toml changes:**
- Removed: `dkls23-ll` (git dependency, v1.1.4)
- Added: `sl-dkls23 = { version = "1.0.0-beta", features = ["test-support"] }`
- Added: `sl-mpc-mate = { version = "1.0.0-beta", features = ["simple-relay"] }`
- Added: `sl-oblivious = "=1.0.0-beta"` (pinned)
- Added: `tokio = { version = "1", features = ["rt", "rt-multi-thread", "sync", "macros"] }`
- Added: `futures-util = { version = "0.3", default-features = false, features = ["sink"] }`
- Added: `rand_chacha = "0.3"`

**rust/src/runtime.rs:** `OnceCell<Runtime>` singleton via `get_runtime()` — lazily initializes a `current_thread` tokio runtime with `enable_all()`. Process-level lifetime, never destroyed.

**rust/src/relay.rs:** `ChannelRelayConn { rx: mpsc::Receiver<Vec<u8>>, tx: mpsc::UnboundedSender<Vec<u8>> }` — implements `Stream<Item=Vec<u8>>`, `Sink<Vec<u8>, Error=MessageSendError>`, `Unpin`, and `Relay` (sl-mpc-mate blanket trait alias). This is the FFI/async bridge: FFI pushes server messages via `tx_in`, reads client messages via `rx_out`.

### Task 2: session.rs + lib.rs

**rust/src/session.rs:** Three session structs rewritten from dkls23-ll state-machine to channel-based:
- `KeygenSession`: `tx_in`, `rx_out`, `task_handle: Option<JoinHandle<Result<Vec<u8>, String>>>`
- `SignSession`: `tx_in`, `rx_out`, `task_handle`, `digest`, `consumed` (SEC-01), `public_key_hex`
- `RecoverySession`: `tx_in`, `rx_out`, `task_handle`, `created_at` (TTL), `current_rotation_version`
- Static maps and `EXPORTED_KEYS` registry preserved

**rust/src/lib.rs:** Added `pub mod relay;` and `pub mod runtime;`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Removed `default-features = false` from sl-dkls23**
- **Found during:** Task 1 cargo check
- **Issue:** Plan specified `default-features = false, features = ["test-support"]` for sl-dkls23. The `test-support` feature's keygen/utils.rs uses `tokio::task::JoinSet`, which requires `tokio/rt-multi-thread`. This is gated behind the `multi-thread` default feature — disabling defaults broke the build.
- **Fix:** Removed `default-features = false` to retain the `multi-thread` feature; also added `rt-multi-thread` to our tokio dependency.
- **Files modified:** rust/Cargo.toml
- **Commit:** 00ff829

**2. [Rule 1 - Bug] Pinned sl-oblivious to =1.0.0-beta**
- **Found during:** Task 1 cargo check
- **Issue:** Cargo resolved `sl-oblivious = "1.0.0-beta"` (from sl-dkls23) to latest `1.1.0`. In 1.1.0, `DLogProof` gained a generic parameter `<C: CurveArithmetic>`, causing E0107 in sl-dkls23's source code (22 compile errors).
- **Fix:** Added `sl-oblivious = "=1.0.0-beta"` as explicit dependency to force exact version.
- **Files modified:** rust/Cargo.toml
- **Commit:** 00ff829

**3. [Rule 1 - Bug] ChannelRelayConn Sink::Error = MessageSendError (not Never)**
- **Found during:** Task 1 relay.rs implementation
- **Issue:** Plan suggested `futures_util::never::Never` as Sink error type. The actual sl-mpc-mate 1.0.0-beta Relay trait requires `Sink<Vec<u8>, Error = MessageSendError>` exactly.
- **Fix:** Used `sl_mpc_mate::coord::MessageSendError` as Sink error type; added explicit `impl Relay for ChannelRelayConn {}`.
- **Files modified:** rust/src/relay.rs
- **Commit:** 00ff829
