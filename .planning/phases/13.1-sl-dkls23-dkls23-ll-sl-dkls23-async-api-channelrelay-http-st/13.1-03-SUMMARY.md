# Phase 13.1 Plan 03: Integration Test Rewrite for sl-dkls23 Summary

**One-liner:** Rewrote all 4 integration test files to use sl-dkls23 async API with SimpleMessageRelay for in-process 2-party simulation, replacing dkls23-ll synchronous 4-round state machine tests.

## What Was Built

### Task 1: test_dkg.rs and test_dsg.rs

**test_dkg.rs:**
- `pub async fn run_dkg_two_party()` — reusable 2-of-2 DKG helper via `keygen::run` + `SimpleMessageRelay`
- `test_dkg_two_party_produces_matching_keyshares` — both parties converge on same public key
- `test_dkg_keyshare_serialization_roundtrip` — `as_slice()` → `from_bytes()` preserves public key
- `test_dkg_evm_address_derivable` — 65-byte uncompressed point → valid 42-char EVM address

**test_dsg.rs:**
- `test_dsg_two_party_produces_valid_signature` — 2-party `sign::run` produces identical signatures; `trial_recovery_from_prehash` verifies recovery ID
- `test_dsg_ecrecover_restores_public_key` — full ecrecover roundtrip restores original VerifyingKey; r and s each 32 bytes
- `test_dsg_consumed_session_rejected` — SEC-01: inserts `SignSession { consumed: true }` into SIGN_SESSIONS, verifies rejection logic

### Task 2: test_rotation.rs and test_backup_export.rs

**test_rotation.rs:**
- `test_rotation_two_party` — `key_refresh::run` with `KeyshareForRefresh::from_keyshare` preserves public key (PROTO-03)
- `test_rotation_new_share_can_sign` — rotated keyshares produce valid DSG signature recoverable to original public key
- `test_session_ttl_eviction` — SEC-02: inserts `RecoverySession` with backdated `created_at`, calls `recover_continue`, verifies "expired" error and session removal

**test_backup_export.rs:**
- `test_backup_roundtrip` — AUX-01: `as_slice()` → Base64 → encrypt → decrypt → Base64 decode → `from_bytes()` → same public key
- `test_backup_wrong_secret_fails` — wrong secret returns Err
- `test_backup_truncated_payload` — truncated payload returns Err without panic
- `test_export_private_key` — AUX-02: `export_private_key(b64_0, b64_1)` reconstructs correct private key and EVM address
- `test_export_blocks_signing` — T-13.1-04: EXPORTED_KEYS contains compressed public key hex after export
- `test_export_invalid_input` — invalid Base64 returns Err

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] tokio::test requires multi_thread flavor**
- **Found during:** Task 1 initial test run
- **Issue:** All 3 DKG tests panicked: "can call blocking only when running on the multi-threaded runtime" (sl-dkls23 dkg.rs:228 internally calls `spawn_blocking`)
- **Fix:** Changed all `#[tokio::test]` to `#[tokio::test(flavor = "multi_thread")]` in all 4 test files
- **Files modified:** all 4 test files
- **Commit:** bde3661, 5057ca1

**2. [Rule 1 - Bug] SimpleMessageRelay::connect() takes no arguments**
- **Found during:** Task 1 implementation
- **Issue:** PLAN.md documented `relay.connect(2)` (connect with party count). Actual API: `SimpleMessageRelay::connect(&self)` — no arguments, returns one `MessageRelay` per call. Call once per party.
- **Fix:** Used `coord.connect()` twice (once per party) instead of `coord.connect(2)`
- **Files modified:** all 4 test files
- **Commit:** bde3661

**3. [Rule 1 - Bug] SEC-01/SEC-02 tests redesigned for new channel-based session structure**
- **Found during:** Task 1 (test_dsg.rs) and Task 2 (test_rotation.rs) implementation
- **Issue:** Old tests used `DsgState`, `DkgState` fields in `SignSession`/`RecoverySession`. New sl-dkls23 sessions have `tx_in: mpsc::Sender`, `rx_out: mpsc::UnboundedReceiver`, `task_handle` — no protocol state fields.
- **Fix:** SEC-01 test inserts a dummy `SignSession` with `consumed=true` using dummy channel pairs. SEC-02 test inserts `RecoverySession` with backdated `created_at` using dummy channel pairs; calls `recover_continue` which hits TTL check before any channel operation.
- **Files modified:** rust/tests/test_dsg.rs, rust/tests/test_rotation.rs
- **Commit:** bde3661, 5057ca1

**4. [Rule 1 - Bug] MessageDigest::from_hex requires exactly 64 hex chars**
- **Found during:** Task 1 SEC-01 test
- **Issue:** Initial hex string in test had 68 chars (34 bytes) causing `from_hex` to return Err("message digest must be exactly 32 bytes")
- **Fix:** Used `'ab'*31 + 'cd'` = exactly 64 chars
- **Files modified:** rust/tests/test_dsg.rs
- **Commit:** bde3661
