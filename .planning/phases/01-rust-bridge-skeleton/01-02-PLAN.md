<threat_model>
## Phase 1 Plan 02 安全威胁模型

1. **DTO 类型安全** — LOW
   - 威胁: Dart DTO fromJson 可能接受畸形 JSON 导致运行时崩溃
   - 缓解: fromJson 工厂方法使用 null-safe 字段访问，MpcRoundResult.fromJson 对必需字段做非 null 断言
   - 验证: 测试中包含 error handling 用例

2. **MpcTransport 接口暴露面** — LOW
   - 威胁: MpcTransport 定义可能泄漏内部协议细节
   - 缓解: MpcTransport 仅定义 `send(endpoint, payload) → response`（per D-07），不暴露 round 数量或协议时序
   - 验证: MpcTransport 接口仅含一个 send 方法

3. **MpcEngine 不暴露给宿主** — LOW
   - 威胁: MpcEngine 作为内部层（per D-06）不应被宿主直接使用
   - 缓解: flutter_mpc_wallet.dart 导出 MpcEngine 供包内使用（后续 MpcClient 使用），文档标注为内部 API
   - 验证: MpcEngine 类文档注释标注 @internal 或等效说明
</threat_model>

    **Step 1: 创建 lib/src/dto/mpc_dtos.dart**

    定义以下 DTO 类（per D-09 对齐架构文档字段草案）:

    `MpcRoundResult` — Rust stub 函数返回的 round 级结果:
    - `String status` (值域: "continue" / "completed" / "error")
    - `int round`
    - `String? clientPayload`
    - `String? errorMessage`
    - `factory MpcRoundResult.fromJson(Map<String, dynamic> json)` — 使用 `json['status'] as String` 等强类型访问
    - `Map<String, dynamic> toJson()`
    - `bool get isContinue => status == 'continue'`
    - `bool get isCompleted => status == 'completed'`
    - `bool get isError => status == 'error'`

    `KeygenResult` — keygen 完成后服务端返回的完整结果（per 架构文档 0.7）:
    - `String mpcKeyId`
    - `String address`
    - `String publicKey`
    - `String curve`
    - `int threshold`
    - `String keyRef`
    - `String backupState`
    - `int rotationVersion`
    - `String localEncryptedShare`
    - `String? encryptedBackupShare`
    - `factory KeygenResult.fromJson(Map<String, dynamic> json)`

    `RecoveryResult` — recovery 完成后服务端返回的完整结果:
    - `String mpcKeyId`
    - `String address`
    - `String publicKey`
    - `int rotationVersion`
    - `String localEncryptedShare`
    - `String? encryptedBackupShare`
    - `factory RecoveryResult.fromJson(Map<String, dynamic> json)`

    `SignResult` — sign 完成后服务端返回的完整结果:
    - `String? signature`
    - `String? signedTx`

    **Step 3: 创建 lib/src/bridge/mpc_engine.dart**

    Per D-06: MpcEngine 是 Rust FFI wrapper 层，对内使用，不暴露给宿主。

    ```dart
    import 'dart:convert';
    import '<path-to-frb-generated>/frb_generated.dart'; // 实际路径以 FRB 输出为准
    import '../dto/mpc_dtos.dart';
    ```

    `MpcEngine` 类:
    - 构造函数: 接收 FRB 生成的 API 抽象类实例（具体类名以 `lib/src/rust/frb_generated.dart` 中实际生成为准，通常为 `RustLibApi`）。使用构造函数注入而非全局单例，便于测试 mock。
    - 6 个异步方法，每个调用对应的 FRB 生成函数并反序列化 JSON 结果:

    `Future<MpcRoundResult> keygenStart(String sessionId, String serverPayload)`:
    1. 调用 FRB API 的 keygen_start 方法（camelCase 版本）
    2. JSON decode 返回的 String
    3. 返回 `MpcRoundResult.fromJson(jsonDecode(result))`

    `Future<MpcRoundResult> keygenContinue(String sessionId, String serverPayload)` — 同上模式

    `Future<MpcRoundResult> recoverStart(String sessionId, String backupShare, String serverPayload)` — 注意多一个 backupShare 参数

    `Future<MpcRoundResult> recoverContinue(String sessionId, String serverPayload)`

    `Future<MpcRoundResult> signStart(String sessionId, String share, String serverPayload)` — 注意多一个 share 参数

    `Future<MpcRoundResult> signContinue(String sessionId, String serverPayload)`

    错误处理: 如果 FRB 调用返回 Err (Rust Result::Err)，FRB 会抛出异常。MpcEngine 方法应将此异常包装为 `MpcRoundResult(status: 'error', round: 0, errorMessage: e.toString())`，或直接 rethrow（由后续 MpcClient 层处理）。Phase 1 选择 rethrow，保持 Engine 层薄。
