Replace the stub `keygen_start` and `keygen_continue` in mpc_engine.rs with real implementations:

**keygen_start(session_id, server_payload):**
1. Parse `server_payload` as JSON WireEnvelope, extract payload, decode CBOR Base64 to `KeygenMsg1`
2. Create `State::new(Party { ranks: vec![0, 0], party_id: 0, t: 2 }, &mut rng, None)`
3. Call `state.generate_msg1()` to get own msg1
4. Call `state.handle_msg1(&mut rng, vec![server_msg1])` to get `Vec<KeygenMsg2>` (1 element, to_id=1)
5. Store `KeygenSession { state, round: 2, my_commitment_2: None, server_commitment_2: None, pending_msg3: None }` into `KEYGEN_SESSIONS`
6. Encode msg2[0] as CBOR Base64, wrap in `WireEnvelope::new(session_id, Dkg, 2, 0, Some(1), payload, None)`
7. Return `MpcRoundResult { status: "in_progress", round: 2, client_payload: Some(envelope_json), error_message: None }`

**keygen_continue(session_id, server_payload):**
1. Parse server_payload as WireEnvelope
2. Lock KEYGEN_SESSIONS, get mutable session. If not found, return Err.
3. Dispatch based on `session.round`:

   **round == 2** (server sends KeygenMsg2):
   - Decode payload to KeygenMsg2
   - Call `session.state.handle_msg2(&mut rng, vec![server_msg2])` -> `Vec<KeygenMsg3>`
   - Call `session.state.calculate_commitment_2()` -> `[u8; 32]`, store in `session.my_commitment_2`
   - Encode KeygenMsg3[0] as CBOR Base64, store in `session.pending_msg3` (cached for Round 3b)
   - Set `session.round = 3`
   - Encode commitment_2 bytes as CBOR Base64, wrap in WireEnvelope(round=3, from_id=0, to_id=None, step=Some("commitment"))
   - Return MpcRoundResult { status: "in_progress", round: 3, client_payload: commitment envelope }

   **round == 3 and envelope.step == Some("commitment")** (server sends commitment_2):
   - Decode payload to `[u8; 32]`, store in `session.server_commitment_2`
   - Retrieve `session.pending_msg3` (the cached KeygenMsg3 CBOR Base64)
   - Wrap cached msg3 in WireEnvelope(round=3, from_id=0, to_id=Some(1), step=Some("msg3"))
   - Return MpcRoundResult { status: "in_progress", round: 3, client_payload: msg3 envelope }

   **round == 3 and envelope.step == Some("msg3")** (server sends KeygenMsg3):
   - Decode payload to KeygenMsg3
   - Build commitment_2_list: `vec![session.my_commitment_2.unwrap(), session.server_commitment_2.unwrap()]` (index 0 = party0, index 1 = party1)
   - Call `session.state.handle_msg3(&mut rng, vec![server_msg3], &commitment_2_list)` -> KeygenMsg4
   - Set `session.round = 4`
   - Encode msg4 as CBOR Base64, wrap in WireEnvelope(round=4, from_id=0, to_id=None, step=None)
   - Return MpcRoundResult { status: "in_progress", round: 4, client_payload: msg4 envelope }

   **round == 4** (server sends KeygenMsg4):
   - Decode payload to KeygenMsg4
   - Remove session from KEYGEN_SESSIONS (take ownership of state)
   - Call `state.handle_msg4(vec![server_msg4])` -> Keyshare
   - Extract public_key: `use k256::elliptic_curve::sec1::ToEncodedPoint; let encoded = keyshare.public_key.to_encoded_point(false); let bytes = encoded.as_bytes();`
   - Call `crate::api::address::derive_evm_address(bytes)` -> EVM address
   - Serialize Keyshare: `serde_json::to_string(&keyshare)` -> local_encrypted_share
   - Build KeygenCompletedPayload: mpc_key_id=session_id, address=evm_address, public_key=hex::encode(bytes), curve="secp256k1", threshold=2, key_ref=session_id, backup_state="none", rotation_version=1, local_encrypted_share=keyshare_json
   - Return MpcRoundResult { status: "completed", round: 4, client_payload: Some(serde_json::to_string(payload)), error_message: None }

**Error handling:** All dkls23-ll errors map to String via `.map_err(|e| e.to_string())`. On error, remove session from KEYGEN_SESSIONS to prevent leak.

**CRITICAL anti-patterns to avoid:**
- Do NOT pass own msg1 to handle_msg1 (only pass OTHER party's msg1)
- Do NOT forget commitment_2_list must have 2 elements indexed by party_id
- Do NOT call calculate_commitment_2 before handle_msg2
- Use `&mut rand::thread_rng()` for rng parameter
