**步骤 3 — 新增私有辅助函数（在 backup 函数之前，在 sign 实现之后）：**

```rust
/// 从 userBackupSecret 派生 32 字节 AES-256 密钥（HKDF-SHA256）。
/// info 字符串绑定应用上下文，防止密钥混用。
fn derive_aes_key(user_backup_secret: &str) -> [u8; 32] {
    let hk = Hkdf::<Sha256>::new(None, user_backup_secret.as_bytes());
    let mut key = [0u8; 32];
    hk.expand(b"ceres-mpc-backup-v1", &mut key)
        .expect("32 bytes is valid HKDF-SHA256 output length");
    key
}

/// 加密明文，返回 hex(nonce || ciphertext_with_tag)。
/// 每次调用随机生成 12 字节 nonce（OsRng），防止 GCM nonce 复用。
fn encrypt_share(plaintext: &[u8], key_bytes: &[u8; 32]) -> Result<String, String> {
    let key = Key::<Aes256Gcm>::from_slice(key_bytes);
    let cipher = Aes256Gcm::new(key);
    let nonce = Aes256Gcm::generate_nonce(&mut OsRng); // 12 bytes
    let ciphertext = cipher
        .encrypt(&nonce, plaintext)
        .map_err(|e| format!("aes-gcm encrypt failed: {e}"))?;
    let mut combined = nonce.to_vec();
    combined.extend_from_slice(&ciphertext);
    Ok(hex::encode(combined))
}

/// 解密 hex(nonce || ciphertext_with_tag)，返回明文字节。
/// GCM tag 验证失败（错误密钥或篡改）返回 Err。
fn decrypt_share(payload_hex: &str, key_bytes: &[u8; 32]) -> Result<Vec<u8>, String> {
    let combined = hex::decode(payload_hex)
        .map_err(|e| format!("hex decode failed: {e}"))?;
    if combined.len() < 12 {
        return Err("payload too short: must be at least 12 bytes (nonce)".to_string());
    }
    let (nonce_bytes, ciphertext) = combined.split_at(12);
    let nonce = Nonce::from_slice(nonce_bytes);
    let key = Key::<Aes256Gcm>::from_slice(key_bytes);
    let cipher = Aes256Gcm::new(key);
    cipher
        .decrypt(nonce, ciphertext)
        .map_err(|_| "aes-gcm decrypt failed: wrong key or corrupted payload".to_string())
}
```

**步骤 4 — 替换 derive_backup_envelope stub：**

```rust
/// Derive a backup envelope from a live share and user secret.
/// Uses AES-256-GCM with HKDF-SHA256 key derivation (per D-01).
///
/// # Arguments
/// - local_encrypted_share: MasterKey2 JSON (plaintext to encrypt)
/// - user_backup_secret: high-entropy secret from host; HKDF-derived to AES-256 key
/// - created_at: ISO-8601 timestamp supplied by caller (Dart layer)
///
/// # Returns
/// JSON-serialized BackupEnvelope with algorithm = "aes-256-gcm-hkdf-sha256"
pub fn derive_backup_envelope(
    local_encrypted_share: String,
    user_backup_secret: String,
    created_at: String,
) -> Result<String, String> {
    let key = derive_aes_key(&user_backup_secret);
    let payload = encrypt_share(local_encrypted_share.as_bytes(), &key)?;
    let envelope = BackupEnvelope {
        version: "1".to_string(),
        algorithm: "aes-256-gcm-hkdf-sha256".to_string(),
        created_at,
        payload,
    };
    serde_json::to_string(&envelope).map_err(|e| e.to_string())
}

/// Decrypt a backup envelope to recover the device backup share.
/// Inverse of derive_backup_envelope (per D-02).
///
/// # Arguments
/// - encrypted_envelope: full BackupEnvelope JSON (not just payload field)
/// - user_backup_secret: same secret used during derive_backup_envelope
///
/// # Returns
/// JSON-serialized DecryptBackupResult{device_backup_share: <original MasterKey2 JSON>}
pub fn decrypt_backup_share(
    encrypted_envelope: String,
    user_backup_secret: String,
) -> Result<String, String> {
    let envelope: BackupEnvelope = serde_json::from_str(&encrypted_envelope)
        .map_err(|e| format!("invalid BackupEnvelope JSON: {e}"))?;
    let key = derive_aes_key(&user_backup_secret);
    let plaintext_bytes = decrypt_share(&envelope.payload, &key)?;
    let device_backup_share = String::from_utf8(plaintext_bytes)
        .map_err(|e| format!("decrypted bytes are not valid UTF-8: {e}"))?;
    let result = DecryptBackupResult { device_backup_share };
    serde_json::to_string(&result).map_err(|e| e.to_string())
}
```

**注意：**
- derive_backup_envelope 签名新增第三个参数 `created_at: String`（Pattern 4 方式 A）
- 不要把 nonce 分开存储，必须前置进 payload hex
- OsRng 引用路径：`aes_gcm::aead::OsRng`，若编译失败改为添加 `rand = "0.8"` 并用 `rand::rngs::OsRng`
- decrypt_backup_share 第一步必须 serde_json::from_str 整个 BackupEnvelope，不要直接解析 payload 字段

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dart → Rust FFI | user_backup_secret 从 host 传入，Rust 不验证熵值，只用于 HKDF |
| Rust → BackupEnvelope storage | payload 离开 Rust 后明文传给 host；host 负责安全存储（MPC-06） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | BackupEnvelope.payload | mitigate | AES-GCM tag 验证：任何字节篡改导致 decrypt_share 返回 Err，Rust 层 propagate 为 Err(String)，Dart 层抛 MpcProtocolException |
| T-05-02 | Information Disclosure | derive_aes_key 中间密钥 | accept | key_bytes 为栈变量，函数返回后即释放；Rust 不 zeroize（无 zeroize crate），可接受：key 生命周期极短 |
| T-05-03 | Repudiation | nonce 复用 | mitigate | 每次调用使用 OsRng 生成随机 nonce；不持久化 nonce；GCM nonce 复用为 catastrophic，OsRng 提供充足熵 |
| T-05-04 | Elevation of Privilege | rotation_version 硬编码 | mitigate | Task 2 修复：从 session 读取并 +1，防止多次 recovery 后版本停在 2 导致 metadata 不一致 |
