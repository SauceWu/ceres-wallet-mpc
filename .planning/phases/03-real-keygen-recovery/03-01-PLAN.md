**3. Create rust/src/api/address.rs** — EVM address derivation from group public key:

```rust
use tiny_keccak::{Hasher, Keccak};

/// Derive an EIP-55 checksummed EVM address from an uncompressed secp256k1 public key.
///
/// Input: 65-byte uncompressed public key (0x04 prefix + 64 bytes X,Y coordinates)
/// Process: Keccak-256 hash of the 64 bytes (skip 0x04) -> take last 20 bytes -> EIP-55 checksum
///
/// Per MPC-07: address derived from keygen protocol's group public key.
pub fn derive_evm_address(uncompressed_pubkey: &[u8]) -> Result<String, String> {
    if uncompressed_pubkey.len() != 65 || uncompressed_pubkey[0] != 0x04 {
        return Err("Expected 65-byte uncompressed public key with 0x04 prefix".to_string());
    }

    // Keccak-256 of the 64 bytes (skip 0x04 prefix)
    let mut hasher = Keccak::v256();
    hasher.update(&uncompressed_pubkey[1..]);
    let mut hash = [0u8; 32];
    hasher.finalize(&mut hash);

    // Take last 20 bytes as address
    let address_bytes = &hash[12..];
    let hex_addr = hex::encode(address_bytes);

    // EIP-55 checksum: keccak256 of lowercase hex, use bit to determine case
    let mut checksum_hasher = Keccak::v256();
    checksum_hasher.update(hex_addr.as_bytes());
    let mut checksum_hash = [0u8; 32];
    checksum_hasher.finalize(&mut checksum_hash);

    let mut checksummed = String::with_capacity(42);
    checksummed.push_str("0x");
    for (i, c) in hex_addr.chars().enumerate() {
        let nibble = (checksum_hash[i / 2] >> (if i % 2 == 0 { 4 } else { 0 })) & 0xf;
        if nibble >= 8 {
            checksummed.push(c.to_ascii_uppercase());
        } else {
            checksummed.push(c);
        }
    }

    Ok(checksummed)
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dart FFI -> Rust | Untrusted JSON payloads cross into Rust crypto functions |
| Server -> Client | Server messages (Party1) parsed and used in crypto protocol |
| Session Map | In-memory state holding cryptographic intermediate values |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation |
|-----------|----------|-----------|----------|-------------|------------|
| T-03-01 | Tampering | server_payload JSON | high | mitigate | Validate all server_payload JSON before passing to kms protocol functions. Use serde_json typed deserialization (not Value) for protocol messages. Protocol's own ZKP verification catches tampered cryptographic payloads. |
| T-03-02 | Information Disclosure | Session Map | high | mitigate | Session map stores EcKeyPair (private material) only in memory, never serialized to disk. Sessions removed immediately after use (keygen_continue/recover_continue). Implement session timeout or cleanup on error paths. |
| T-03-03 | Spoofing | session_id | medium | mitigate | Session ID must exist in map for continue to succeed — random guessing is infeasible with UUID-length IDs. Return generic error "session not found" without leaking session state. |
| T-03-04 | Denial of Service | Session Map | medium | mitigate | Unbounded session creation could exhaust memory. Add a reasonable max session count (e.g., 100) and reject new sessions when full. Clean up abandoned sessions. |
| T-03-05 | Information Disclosure | localEncryptedShare | high | mitigate | MasterKey2 serialized as JSON contains private key material. This blob MUST be treated as secret — Dart side stores in secure storage only. Rust side never logs this value. Debug/Display impls must redact. |
| T-03-06 | Tampering | EVM address derivation | high | mitigate | Address derived deterministically from group public key using Keccak-256. Unit test verifies known test vector. Client may cross-check against server-provided address. |
</threat_model>
